Agent / Experiment Loop

High-level view of the LLM data-scientist loop and the 003 memory-compaction ablation (Conditions A / B / C).

One experiment iteration (repeat for N iterations, per dataset × condition × seed)

STEP 1
Build context
Dataset summary, current best run, allowed actions + memory per A/B/C.
STEP 2
LLM proposes
Structured JSON: action, model family, hyperparameters, rationale.
STEP 3
Validate & train
Check allowlist + params, run sklearn model, CV score (RMSE/R²/MAE).
STEP 4
Score vs incumbent
Accept the run if the primary metric improves; update best run.
STEP 5
Log run
Append prompt, visible memory IDs, tokens, action, metrics to history.
↺
Logged history & updated best run feed Step 1 of the next iteration

The manipulated variable — memory shown to the agent at Step 1

Everything else is frozen (same LLM, prompt, model registry, metric, dataset, seeds, budget). Only what memory the agent sees changes between conditions.

CONDITION A
Recent-only
Last k raw experiment records only. Short working-memory baseline.
CONDITION B
All-raw
Every prior raw record, appended forever. Tests context-overload harm.
CONDITION C
Compacted + recent
Latest structured compaction artifact + last k raw records.
Compaction trigger (Condition C): every m iterations, the LLM compactor reads past runs and emits a structured artifact — confirmed findings, failed/promising directions, best-known configs, open questions, next-step recommendation. It never sees future outcomes (no leakage) and is reused unchanged until the next compaction point.
