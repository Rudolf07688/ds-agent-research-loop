What 004 delivers

  • A versioned suite of 5 tabular members — regression and classification — each with a frozen train/val/test split, a pre-registered primary metric + direction, a fixed experiment budget, a frozen action space, and a task-appropriate model allowlist. Composition: delivery_time (anchored-synthetic, regression) + diabetes (regression) + wine / iris / breast_cancer (classification).
  • Provenance is mixed: anchored-synthetic (deterministic Python generation, no LLM) and curated-real sklearn datasets for external validity. The seed delivery-time task is now one member, not the whole project.
  • Frozen split rows are materialized and persisted in Postgres (Alembic-managed) and exportable to JSON/CSV, byte-identical across every regime and seed (SC-001…SC-007 all met).
  • Scope held to the data + harness layer only — no memory regimes (005), no compaction operator (006), no ablation study (007).

How it was built

  • Foundational: DatasetDescriptor extended with provenance / budget / patience / action_space / model_allowlist (+ task-type validation); FrozenSplit, content_hash, member_fingerprint, and BenchmarkDriftError / SplitError added to benchmark.py.
  • Persistence: benchmark_members + benchmark_splits tables (SQLAlchemy Core + FakeStore mirror) created exclusively via Alembic migration 0002 (verified upgrade → downgrade → upgrade, idempotent & reversible).
  • API: materialize_suite (idempotent, drift-rejecting), load_member (content-hash assertion), check_version_drift (version coexistence), export_member (byte-identical JSON/CSV round-trip), plus a benchmark CLI (materialize / export).
  • Splits: stratified two-stage split for classification (every class in every partition); fixed-seed shuffle for regression; no test-row leakage.
  • Loop (US4): main / experiment sweep now resolve the descriptor + frozen split from the materialized suite via load_member; the frozen action space is enforced and the stop reason is recorded — no dataset-specific path.
  • Tests: 106 passing (offline + Postgres integration), covering descriptor validation, hash stability, stratification, version drift/coexistence, export round-trip, and re-materialization idempotency (SC-001…SC-007).
