What was built

  • prompts.py — Pydantic entities, the single pydantic-settings Settings object, both LLM JSON schemas, and prompt text (the centralized config home).
  • llm.py — async OpenAI-compatible client + generic schema-constrained, Pydantic-validated request helper; the two sanctioned calls (seed, next-step).
  • data_gen.py — seed bootstrap with checkpoint resume; local dataset expansion anchored to the saved spec (no LLM, no spec regeneration).
  • train.py — model allowlist, hyperparameter + decision validation (the safety boundary), one-hot pipeline, 5-fold CV scoring (RMSE/R²/MAE).
  • history.py — append-only run history, best-run tracking (RMSE-only improvement), rejection recording that retains the prior model.
  • main.py — CLI + orchestration: seed → expand → score → next-step → record; baseline LinearRegression first; stop at N or after patience rounds.
  • tests/ — 20 offline pytest units across data-gen, train, history, loop.

Key decisions

  • Flat repo-root layout (replaced the generated src/ package) and requires-python >= 3.11, per the plan and Constitution Principle I.
  • Settings + all Pydantic models live in prompts.py — honours the fixed module list, avoids an unsanctioned config.py (open item C1).
  • Hyperparameter values are checked via scikit-learn's _validate_params() so invalid values are rejected before training (1.9 validates at fit time).
  • Rejections carry the retained model's metrics (no Infinity in JSON), keeping history.json standard and inspectable.
